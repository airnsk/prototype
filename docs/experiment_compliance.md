## 1. Топология стендов

Реализованы три изолированных Docker-стенда в директории `stands/`:

| Стенд          | Клиенты        | Сервисы | Инфраструктурный узел               | Сети                                              |
| ------------------- | --------------------- | -------------- | ------------------------------------------------------- | ----------------------------------------------------- |
| `stands/direct/`  | 1 (`direct-client`) | 3 (S1, S2, S3) | `direct-controller`, `direct-server` (server-agent) | control-plane 172.22.0.0/24, data-plane 172.23.0.0/24 |
| `stands/gateway/` | 1 (`gateway-probe`) | 3 (S1, S2, S3) | `gateway-ctrl`, `gateway-ztna` (шлюз)           | control-plane 172.26.0.0/24, data-plane 172.27.0.0/24 |
| `stands/vpn/`     | 1 (`vpn-client`)    | 3 (S1, S2, S3) | `vpn-server` (OpenVPN)                                | control-plane 172.24.0.0/24, data-plane 172.25.0.0/24 |

Все стенды изолированы друг от друга (разные Docker-сети).

## 6. Выявленные ограничения

### Архитектурные

1. **Масштаб стенда** — физические измерения выполнены на стенде (1×3) в контейнерной среде. Масштабы (10, 5), (100, 50), (1000, 500) оценены расчётным путём.
2. **Единый server-agent** — в Direct-ZTNA один server-agent обслуживает все сервисы. В промышленном развёртывании каждый сервис должен иметь свой server-agent для полной изоляции.
3. **Gateway-ZTNA** — шлюз проксирует по plain HTTP внутри data-plane (упрощение прототипа).

### Экспериментальные

4. **NTP-синхронизация** — не реализована, не критична для Docker-стенда.
5. **Телеметрия VPN и Gateway-ZTNA** — логи собираются, но не парсятся в едином формате MetricEvent.
6. **Криптостойкость** — не являлась предметом анализа; используются Ed25519 и TUN-шифратор AES-256-GCM.

## 7. Как запустить эксперименты

### Эксперимент экспонированности (exposure)

```bash
cd stands/gateway
sg docker -c "docker-compose up -d"
python3 exposure_experiment.py
sg docker -c "docker-compose down"
```

Аналогично для `stands/vpn/` и `stands/direct/`.

### Полный benchmark (T_setup, T_revoke, RTT)

```bash
cd stands/<mode>
sg docker -c "docker-compose up -d"
python3 experiment.py        # 10 прогонов
sg docker -c "docker-compose down"
```

Или запуск всех трёх стендов через `run_benchmarks.py`:

```bash
python3 stands/run_benchmarks.py
```
